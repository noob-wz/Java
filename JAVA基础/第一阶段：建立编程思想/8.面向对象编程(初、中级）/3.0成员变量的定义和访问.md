# 🧱 零基础拆解：什么是成员变量？

首先，我们要明确你在代码里写的位置。

打开你的 Java 代码（类），你会看到一对大括号 `{ ... }`。

- 写在**方法里面**的（比如 `main` 方法里），叫**局部变量**（临时工，用完就扔）。
- 写在**类里面，但方法外面**的，就叫**成员变量**（Field/Member Variable）。它们是这个类的核心资产。

而在成员变量这个大家族里，根据**“归属权”**的不同，严格分为两派：

1. **实例变量 (Instance Variable)**
2. **静态变量 (Static Variable)**

我们一个一个来看。

---

### 1. 👤 实例变量 (Instance Variable)

这是最常见的变量。你之前见到的 `String name;` `int age;` 都是这种。

- **标志：** 前面**没有** `static` 关键字。
- **物理归属：** 属于**每一个具体的对象（实例）**。
- **生活类比：** **员工工牌上的“名字”**。
- 张三有张三的名字，李四有李四的名字。
- 每个人的名字是**独立**的，互不干扰。

- **内存位置：** 存放在 **堆内存 (Heap)** 的对象里。每 `new` 一个新对象，就在堆里新开辟一块空间存它。

### 2. 📢 静态变量 (Static Variable)

这是特殊的存在。

- **标志：** 前面**有** `static` 关键字。
- **物理归属：** 属于**整个类（Class）**，不属于某个具体的对象。
- **生活类比：** **公司前台挂着的“公司名称”**。
- 不管你是张三还是李四，咱们的“公司名称”都叫“阿里巴巴”。
- 它只有一份，**全员共享**。如果老板把公司改名叫“字节跳动”，那么张三和李四看到的公司名都会同时变成“字节跳动”。

- **内存位置：** 存放在 **方法区/元空间 (Metaspace)**。无论你 `new` 多少个对象，它在内存里**只有一份**。

---

### 🎨 ASCII 深度图解：内存里的“私有”与“公有”

假设我们有这样一个类：

```java
class Employee {
    // 1. 实例变量：每个人都不一样
    String name;

    // 2. 静态变量：大家公用一份
    static String company = "阿里";
}

```

当你执行 `Employee e1 = new Employee(); e1.name = "张三";` 和 `e2... "李四"` 时：

```text
       【 堆内存 (Heap) 】                     【 方法区 (Method Area) 】
      (员工工位 - 实例变量)                     (公司大厅 - 静态变量)

   +-------------------------+              +-------------------------+
   | 对象 e1 (张三)           |              |  类 Employee            |
   |-------------------------|              |-------------------------|
   | - name: "张三"          |              | - static company: "阿里" |
   +-------------------------+              +-------------------------+
                                                     ^     ^
   +-------------------------+                       |     |
   | 对象 e2 (李四)           |                       |     |
   |-------------------------|                       |     |
   | - name: "李四"          | ----------------------+     |
   +-------------------------+                             |
                                                           |
   (注意：e1 和 e2 虽然住在堆里不同房间，但抬头看的公司名是同一个)

```

### 3. 🔧 点操作符 (`.`) 的真实含义

搞清楚了上面两个概念，我们再来看 **点 (`.`)**。它是用来**访问**这些变量的工具。

但在用法上，针对实例变量和静态变量，有着本质的区别：

#### A. 操控实例变量：必须先有对象

因为实例变量属于“具体的某个人”，所以你必须先 `new` 出来一个人，才能点出他的名字。

- **语法：** `对象引用.变量名`
- **代码：**

```java
Employee e1 = new Employee();
e1.name = "张三"; // 正确：我要改 e1 这个人的名字
// Employee.name = "王五"; // 错误！编译器会骂你：Employee 是图纸，图纸没有名字！

```

#### B. 操控静态变量：推荐用类名

因为静态变量属于“公司”，你不需要找具体的员工，直接找“公司”就能操作。

- **语法：** `类名.变量名` (推荐标准写法)
- **代码：**

```java
Employee.company = "字节"; // 正确：直接把招牌改了

// 此时如果你打印 e1.company，它也变成了 "字节"

```

---

### 🛑 总结一下 (The Checkpoint)

1. **成员变量** = 写在类里、方法外的变量。
2. **实例变量 (无 static)** = 私家车。每人一辆，停在堆内存。用 `对象.` 访问。
3. **静态变量 (有 static)** = 公交车。全村一辆，停在方法区。推荐用 `类名.` 访问。
