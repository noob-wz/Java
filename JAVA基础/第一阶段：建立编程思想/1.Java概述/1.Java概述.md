# 二、Java 重要特性

1. Java 语言是**面向对象**的（oop)
2. Java 语言是**健壮**的。Java 的 _强类型机制、异常处理、垃圾的自动收集等_ 是 Java 程序及安装性的重要保证
3. Java 语言是**跨平台性**的，通常用一句口号来概括 **“一次编写，到处运行”** 。
   - 传统的编程语言（C/C++）编写的代码，**编译器编译后会转换成特定操作系统能理解的机器码**。这意味着 Windows 下编译出的程序，在 Linux 下无法运行。
   - JAVA 的做法是
     - （1）首先，JAVA 编译器**不会直接把源代码编译成机器语言，而是一个中间代码 —— 字节码（`.class`文件）**，它不针对特定的操作系统。
     - （2）然后，**不同的操作系统有不同的 JVM（JAVA 虚拟机），借助 JVM 将中间代码即字节码翻译成对应操作系统能理解的机器码并执行**
4. Java 语言是**解释型**的
   - 解释型语言：javascript, PHP, Java
   - 编译型语言：C, C++
   - 具体区别：
     - **解释型语言编译后的代码不能被机器直接执行，需要解释器来执行**
     - **编译型语言编译后的代码可以直接被机器直接执行**

# 四、Java 开发环境搭建

> JDK 下载地址: https://www.oracle.com/java/technologies/downloads/

**【windows 下安装步骤说明】**

1. **第一步：下载并安装**

- 首先，查看自己计算机时 32 位还是 64 位操作系统，对应下载。安装时，会提供可选功能，主要包含：**开发工具**、**源代码**、**公共 JRE**。
- 当提示**安装 JRE 时，可以选择不安装**
- **建议安装路径为非 C 盘**。比如：D:\program\JDK8。另外，安装路径**不要有中文或者特殊符号如空格等**

2. **第二步：配置 path 环境变量**

- `PATH 环境变量`是操作系统用来**查找可执行文件（如程序或命令）的路径列表**。
- 为什么要配置环境变量 path？
  - 如果不配置环境变量，直接在进入 dos 后 (快捷方式 win+r)敲入 javac，就会出现“错误提示：javac 不是内部或外部命令，也不是可运行的程序或者批处理文件”
  - **错误原因：**
    - 如果不配置 path 环境变量，正常你每次执行命令前，都必须输入冗长的完整路径，例如：未配置时： C:\Program Files\Java\jdk-17\bin\javac MyProgram.java。然后在该目录下正常运行 Java 相关命令，如：`javac`（Java 编译器），`java`（Java 运行时启动器）。实际使用中，每次切换到 JDK 安装路径的\bin 去执行 java 命令，太复杂。而如果在进入到 dos 时的根目录下去执行 直接 java 命令，它是不存在的，系统找不到。此时系统并不会立马报错，系统会在 path 的环境变量中查找这些命令（可执行文件），如果仍未找到，就会出现以上的错误提示。
- **正确配置 path 环境变量的步骤：**
  1. 我的电脑 —— 属性 —— 高级系统设置 —— 环境变量
  2. 增加 JAVA_HOME 环境变量，包含 JDK 根目录路径。如：“变量名：JAVA_HOME”，“变量值：D:\program\JDK8”
     - JAVA_HOME 并非系统必须的环境变量，但它是一个约定俗成的规范，对 Java 开发至关重要，主要**供第三方软件使用**
  3. 编辑 path 环境变量，增加`%JAVA_HOME%\bin`。即将 JDK 安装目录下的 bin 文件夹路径添加到 path 环境变量中。这样，在任何目录下，都可以直接执行 java 命令，如：javac MyProgram.java
     - `%JAVA_HOME%`指的是 JAVA_HOME 变量的变量值
  4. 如果完成以上步骤，在 dos 中输入命令依然报错，重新打开 dos 即可

> **【环境变量配置细节】**
> 当配置环境变量时，准确地说在增加 JAVA_HOME 之前，可以看到环境变量可以配置分为两种:
> (1)一种是**Administrator 的用户变量**下设置
> (2)另一种是配置**系统变量**
>
> 前者只能在该用户使用场景下才能生效，后者是对所有用户都生效的，对应的使用范围更广

# 六、JAVA 开发注意事项和细节说明

1. Java 源文件以`java`为扩展名，它的**基本组成部分是类`(class)`**。可以看到，编译器编译程序时，只要程序中存在多个类，**编译器就会给每个类生成一个对应的.class 文件**
2. Java 应用程序的**执行入口是 `main()`方法**。它有**固定的书写格式**：
   `public static void main(String[] args) {}`
3. Java**严格区分大小写**。也就是说，在 Java 中，字母大写和小写是完全不同的两种字符
4. Java 方法由一条条语句组成，**每个语句以`;`结束**。
5. 大括号都是成对出现的，缺一不可。[习惯，**先写`{}`再写代码**]
6. 一个源文件中**最多只能有一个`public类`**，其他类的个数不限
7. **如果源文件中包含了一个`public类`**，则文件名必须**按照该类名命名**
8. 一个源文件中最多只能有一个`public类`，其他类的个数不限。也**可以将 main 方法写在非 public 类中**，然后指定运行非 public 类，这样入口方法就是非 public 的 main 方法

   ```java
   // 程序中有一个public类型的Hello类，一起其他两个类：Tiger, Dog，并且都包含了main方法
   // 1. 首先，javac Hello.java之后，会生成Hello.class、Tiger.class以及Dog.class
   // 2. 然后，只需要运行对应的类（比如java Dog）就可以执行代码了

   public class Hello {
      public static void main(String[] args) {
         System.out.println("hello, world!");
      }
   }

   class Tiger {
      public static void main(String[] args) {
         System.out.println("hello, tiger!");
      }
   }

   class Dog {
      public static void main(String[] args){
         System.out.pirntln("hello,dog!");
      }
   }
   ```

# 七、Java 转义字符

# 八、Java 开发规范

1. 类、方法的注释，javadoc 的方式来写
2. 非 Java Doc 的注释（也就是单行注释或者多行注释），往往是写给维护者来看的，着重告诉读者为什么这样写，如何修改，注意什么问题等
3. 运算符和 = 两边习惯性地各加一个空格，比如：2+ 4 \* 5 + 345 -24
4. 源文件要使用 utf-8 编码（因为 java 编译器解码后会同意转化为 Unicode 字符）
5. 行宽度不要超过 80 字符
6. 代码编写 **次行风格** 和 **行尾风格**。其实就是大括号是跟在行尾，还是次行

# 九、如何快速学习技术或者知识点

当学习一门新技术时

1. 第一步：**先看看能否使用传统技术解决**，那么有两种结果：能解决，但不完美；不能解决。
2. 第二步：引出我们学习的新技术和知识点。面对传统技术不能解决或者不能完美解决的问题，新的解决方案可能不止一种，此时就要**比较不同的解决方案后确定要选择的新技术**
3. 第三步：确定要学习的新技术后，先学习该技术的**基本原理**和**基本语法**，但**不用考虑细节**
4. 第四步：了解新技术的基本语法和原理之后，**写一个快速入门的案例**
5. 第五步：开始研究技术的注意事项、使用细节、规范和如何优化（也就是说，**一开始学习新技术不要过于追求完美，优化永无止境，容易陷入探究的死循环**）
