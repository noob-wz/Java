### 一、 什么是方法重载？(The Concept)

在同一个类中，允许存在一个以上的**同名方法**，只要它们的**参数列表不同**即可。这就是方法重载。

- **关键词：** 同名、不同参。
- **本质：** 这是一种**编译时多态**（Compile-time Polymorphism）。编译器在编译代码时，会根据你传入参数的类型和数量，自动判断“哦，你想调用的原来是这个版本的代码”，然后绑定上去。

### 二、 生活中的类比 (The Analogy)

想象一下你在餐厅\*\*“买单”\*\*（这个动作就是一个方法）：

1.  你递过去现金 $\rightarrow$ 服务员收现金找零。
2.  你递过去信用卡 $\rightarrow$ 服务员拿 POS 机刷卡。
3.  你拿出手机扫码 $\rightarrow$ 服务员展示二维码。

虽然你的动作都叫 **“买单”**（方法名相同），但因为你**给出的东西不同**（参数类型不同），服务员的**处理方式**（方法体逻辑）也完全不同。这就是重载。

### 三、 核心规则：如何构成重载？ (The Rules)

要构成重载，必须满足以下条件（缺一不可）：

1.  **作用域：** 必须在**同一个类**中。
2.  **方法名：** 必须**完全相同**。
3.  **参数列表：** 必须**不同**。这里的“不同”指以下三种情况之一：
    - **个数不同：** 如 `add(int a)` 和 `add(int a, int b)`
    - **类型不同：** 如 `add(int a)` 和 `add(double a)`
    - **顺序不同：** 如 `add(int a, double b)` 和 `add(double a, int b)` （虽允许，但不推荐，因为容易混淆）

### 四、 设计者视角：为什么要设计重载？ (The Why)

如果没有重载，Java 世界会变得非常可怕。让我们看看 `System.out.println` 这个最常用的方法。

**如果没有重载，你可能需要记住这样的 API：**

- 打印整数：`printlnInt(10)`
- 打印字符串：`printlnString("Hello")`
- 打印浮点数：`printlnFloat(3.14)`

这太痛苦了！为了减轻程序员的记忆负担，我们设计了重载，让你只需要记住一个单词 **`println`**。你把数据扔给它，它自己知道该怎么办。

### 五、 代码示例 (The Code)

让我们来写一个简单的计算器 `Calculator`，体验一下重载的便利：

```java
public class Calculator {

    // 1. 两个整数相加
    public int add(int a, int b) {
        System.out.println("调用了 int, int 版本");
        return a + b;
    }

    // 2. 两个小数相加 (参数类型不同)
    public double add(double a, double b) {
        System.out.println("调用了 double, double 版本");
        return a + b;
    }

    // 3. 三个整数相加 (参数个数不同)
    public int add(int a, int b, int c) {
        System.out.println("调用了 int, int, int 版本");
        return a + b + c;
    }

    public static void main(String[] args) {
        Calculator calc = new Calculator();

        // 编译器会根据参数，自动找到对应的版本
        calc.add(1, 2);       // 自动匹配第 1 个
        calc.add(1.5, 2.5);   // 自动匹配第 2 个
        calc.add(1, 2, 3);    // 自动匹配第 3 个
    }
}
```

### 六、 ⚠️ 唯一的陷阱：返回类型

这是初学者最容易犯的错误，请务必记住：

> **方法重载与“返回类型”无关！** > **方法重载与“访问修饰符”无关！**

**错误示例：**

```java
public int test(int a) { return a; }
public void test(int a) { } // 编译报错！这不是重载，是重复定义。
```

**为什么？**
因为当你调用 `test(10)` 时，编译器无法确定你是想要返回值的那个，还是不想要返回值的那个。为了避免歧义，Java 禁止这种情况。

---

### 总结

- **方法重载** = 同名不同参。
- **目的** = 方便你记忆，让代码更通用、更灵活。
- **判断标准** = 只看**参数列表**（类型、个数、顺序），不看返回值。
