我们可以把 **包 (Package)** 理解为操作系统里的 **“文件夹”**。

---

### 1. 为什么我们需要包？（三大作用）

如果没有包，所有的 Java 类都堆在一起（就像你之前那样），会发生什么？

#### A. 解决命名冲突 (Namespace) —— **最核心的作用**

* **生活场景：** 你们班有两个叫“张三”的同学。老师点名时，大家不知道在叫谁。怎么区分？老师会说：“第一排的张三”和“最后一排的张三”。
* **代码场景：**
  * 你写了一个 `User` 类。 你的同事也写了一个 `User` 类。 你们用的第三方库（比如阿里的库）里也有一个 `User` 类。
* **如果没有包**，编译器就会崩溃：“到底是用哪个 User？”


* **包的解法：**
  * `com.myapp.User` (你的)
  * `com.colleague.User` (同事的)
  * `com.alibaba.User` (阿里的)
* **全名不同，就能共存！**



#### B. 分类管理 (Organization)

* **生活场景：** 你的衣柜里，不会把内裤、外套、袜子乱堆在一起。你会把它们分格存放。
* **代码场景：** 当类很多时，可以很好的管理类：
  * `com.myapp.model`：专门放数据模型（如 User, Order）。
  * `com.myapp.util`：专门放工具类（如 DateUtils）。
  * `com.myapp.controller`：专门放控制逻辑。
* **一句话总结：** 包 **通过‘横向区分业务边界’与‘纵向聚合功能组件’，构建了高内聚、低耦合的代码架构。**



#### C. 访问控制 (Access Protection)

* Java 有一种权限叫 **包级私有 (Package-Private)**（即不写 public 也不写 private）。
  * 这意味着：**同一个包里的兄弟类可以互相访问，但外面的包不行。** 这是一种保护机制。

### 2. 包的语法规则

#### A. 声明包 (Declaring)

这是给类“上户口”。**必须放在 Java 文件的第一行**（注释除外）。

```java
package com.example.learn; // 👈 这就是声明

public class Student {
    // ...
}

```

* **注意：** 代码里的包名，必须和电脑上的**物理文件夹层级**完全对应！
* 上面的代码必须存放在 `src/com/example/learn/Student.java`。



#### B. 导入包 (Importing)

如果你想在 A 类里使用 B 类，而它俩不在同一个包，你就得**导入**。

```java
package com.example.learn;

// 👇 必须显式告诉 Java，这个 List 是 java.util 包里的，不是 java.awt 包里的
import java.util.List; 
import java.util.ArrayList;

// 👇 偷懒写法：导入 java.io 包下的所有类（不推荐滥用，容易乱）
import java.io.*; 

public class Test {
    public static void main(String[] args) {
        List<String> list = new ArrayList<>();
    }
}

```

### 3. 包的命名规范（行业标准）

你在看很多开源代码时，会发现包名都很长，比如 `org.springframework.context`。这是有讲究的。

* **规则 1：全部小写**。不要用 `MyPackage`，要用 `mypackage`。
* **规则 2：域名倒写**。
  * 这是为了保证全球唯一性。因为域名是全球唯一的。
  * 如果百度的域名是 `baidu.com`，他们的包名就会以 `com.baidu` 开头。
  * 如果你的个人网站是 `dchuan.me`，你的包名可以叫 `me.dchuan.projectname`。


* **规则 3：层级清晰**。
* `com.公司名.项目名.模块名`
* 例如：`com.dchuan.blog.user`

---

### 4. 一个特殊的包：`java.lang`

你可能会问：*“为什么我写 `String`, `System.out.println` 的时候，从来不用写 `import`？”*

* **原因：** `java.lang` (Language) 是 Java 的核心包。
* **特权：** Java 编译器会自动、默认地把它导入到每一个文件里。
* **包含成员：** `String`, `System`, `Math`, `Thread`, `Object` 等最基础的类。

除了 `java.lang`，其他的包（包括 `java.util`, `java.io`）都必须手动 import。

### 5. 在 IDEA 中使用包的小技巧

在 IDEA 里建包非常爽，不需要一级一级建文件夹。

1. 右键点击 `src` 文件夹 -> **New** -> **Package**。
2. 输入：`com.dchuan.study.day01`。
3. **回车**。
4. IDEA 会自动帮你创建 `com` -> `dchuan` -> `study` -> `day01` 这四层文件夹。
