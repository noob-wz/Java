在 Java 中，当使用 `new` 关键字动态创建任何数组时（无论是 `int[]`、`Object[]` 还是 `boolean[]`），数组的每个元素都会被自动初始化为其对应类型的默认值。

---

## 💾 Java 数组元素默认值一览表

我们将数组元素的类型分为三大类：**数值类型**、**布尔/字符类型**，和**引用类型**。

### 一、 数值基本数据类型（Numeric Primitives）

所有数值类型（整数和浮点数）的默认值都是 **0**。

| 元素类型 | 数组示例   | 默认值 | 整数值 | 科学/技术说明                  |
| :------- | :--------- | :----- | :----- | :----------------------------- |
| `byte`   | `byte[]`   | $0$    | $0$    | 最小的整数类型，默认为 $0$。   |
| `short`  | `short[]`  | $0$    | $0$    | 默认值为 $0$。                 |
| `int`    | `int[]`    | $0$    | $0$    | 最常用的整数类型，默认为 $0$。 |
| `long`   | `long[]`   | $0L$   | $0$    | 默认为长整型 $0$。             |
| `float`  | `float[]`  | $0.0f$ | $0$    | 单精度浮点数，默认为正 $0.0$。 |
| `double` | `double[]` | $0.0d$ | $0$    | 双精度浮点数，默认为正 $0.0$。 |

### 二、 布尔和字符类型（Boolean & Character）

| 元素类型  | 数组示例    | 默认值   | 整数值 | 科学/技术说明                                      |
| :-------- | :---------- | :------- | :----- | :------------------------------------------------- |
| `boolean` | `boolean[]` | `false`  | $0$    | 布尔类型的默认状态。                               |
| `char`    | `char[]`    | `\u0000` | $0$    | **空字符**（Null Character），Unicode 编码为 $0$。 |

### 三、 引用数据类型（Reference Types）

所有引用数据类型（包括数组本身、类、接口等）的默认值都是 **`null`**。

| 元素类型   | 数组示例   | 默认值 | 内存含义                                                        |
| :--------- | :--------- | :----- | :-------------------------------------------------------------- |
| `String`   | `String[]` | `null` | 尚未指向堆内存中的任何 `String` 对象。                          |
| `Object`   | `Object[]` | `null` | 尚未指向堆内存中的任何对象实例。                                |
| **其他类** | `Car[]`    | `null` | 尚未指向任何 `Car` 类的实例。                                   |
| **数组**   | `int[][]`  | `null` | 主干数组的元素（子数组引用）默认为 `null`，表示尚未创建子数组。 |

---

### 💡 核心原理总结

数组元素的默认值，实际上就是**该数据类型作为类（Class）的成员变量时的默认值**。

当您执行 `new Type[N]` 时：

- **基本类型**：JVM 将对应的内存区域直接填充为该类型的二进制零值（即 $0$ 或 `false`）。
- **引用类型**：JVM 将对应的内存区域填充为表示空引用的值（即 `null`）。
