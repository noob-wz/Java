> 变量的原理就是通过**数据类型**在计算机的内存中**分配（预留）一块特定大小的内存空间**，然后**把这个内存空间的物理地址与变量名建立映射（关联）**。此刻开始，**在代码中使用这个变量名，系统在底层都是使用的对应的那块内存空间的物理地址**。变量名是给人看的，内存地址是给计算机看的

> 变量是**一块特定内存空间**的**抽象** + **命名**
>
> - “抽象”：变量本质就是一块内存空间，只不过隐藏了 **空间大小、二进制细节和地址**；
> - “命名”：变量名和内存地址建立关联。一个易于记忆的名称，通过编译器和操作系统的协作，会映射到对应的物理内存空间。

### 📌 变量的三要素

在像 Java 这样的静态类型语言中，一个变量由三个基本要素构成：

1.  **数据类型 (Type)：**

    - **作用： 可以理解为是变量的使用说明书** 。决定了变量可以
      - **存储什么类型的数据**，不同类型数据的**存储形式** 、**输出显示** 都是不同的
      - 在内存中**分配多大的空间**
      - **允许执行那些操作** （`int` 可以加减乘除，`boolean` 只能进行逻辑运算）。
    - **示例：** `int` (整数), `double` (小数), `char` (字符), `String` (字符串)。

2.  **变量名 (Name)：**

    - **作用：** 您给变量起的名字，用于在程序中**引用**这块内存空间。
    - **示例：** `age`, `score`, `userName`。

3.  **值 (Value)：**

    - **作用：** 实际存储在内存中的数据。
    - **示例：** `25`, `99.5`, `'A'`, `"张三"`。

**示例声明：**

```java
//  数据类型  变量名  =  值;
    int       age     =  25;
    double    price   =  19.99;
    String    name    =  "Java";
```

### ⚙️ 变量的底层原理（内存机制）

深入来看，变量是高级编程语言对计算机**内存（RAM）的一种抽象**。当您声明并使用一个变量时，计算机内部发生了以下几个关键步骤：

#### 1\. 声明与内存分配 (Reservation)

当您在代码中声明一个变量时，例如：
`int score;`

1.  **类型识别：** 编译器（或 JVM）首先看到数据类型是 `int`。
2.  **空间计算：** 它知道 `int` 类型在 Java 中固定占用 **4 个字节 (bytes)** 的内存空间。
3.  **分配地址：** 操作系统会在**内存（RAM）** 中寻找一块连续的、空闲的 4 字节空间。假设它找到了一个起始地址为 `0x00A1F8C4` 的位置。

#### 2\. 映射与命名 (Mapping)

现在，这块内存地址（`0x00A1F8C4`）对于人类来说太难记忆和使用了。

1.  **建立映射：** 编译器会创建一个内部的 **“符号表”** 。
2.  **关联：** 它在这个表中建立一个映射关系：您起的名字 `score` 和 **内存地址（起始地址） `0x00A1F8C4`** 关联起来。

**从这一刻起，您在代码中使用 `score`，系统在底层使用的都是 `0x00A1F8C4` 这个地址。** 变量名是给程序员看的，内存地址是给计算机硬件用的。

#### 3\. 赋值 (Assignment)

当您给变量赋值时，例如：
`score = 100;`

1.  **值转换：** 计算机将数值 `100` 转换为其**二进制**表示形式（`00000000 00000000 00000000 01100100`），这个二进制序列刚好可以存入 4 个字节。
2.  **写入内存：** 操作系统找到 `score` 对应的地址 `0x00A1F8C4`，并将这一串二进制数据**写入**到该地址开始的 4 个字节空间中。

#### 4\. 访问与修改 (Access & Modification)

- **访问 (Read)：** 当您使用 `System.out.println(score);`

  1.  系统查找 `score` 对应的地址 `0x00A1F8C4`。
  2.  它从该地址**读取** 4 个字节的二进制数据。
  3.  根据 `int` 类型规则，将这串二进制数据**解释**为十进制数 `100` 并输出。

- **修改 (Write)：** 当您使用 `score = 50;`

  1.  系统将 `50` 转换为新的二进制数据。
  2.  它找到 `score` 对应的地址 `0x00A1F8C4`。
  3.  用 `50` 的二进制数据**覆盖（Overwrite）** 掉原来存储 `100` 的数据。

### 总结

**变量的原理**，就是通过**数据类型**在内存中**分配（预留）一块特定大小的空间**，然后把**变量名**与这块空间的**物理地址**建立**映射（关联）**。这使得程序员可以使用易于记忆的名称，来完成对复杂内存地址中数据的读写操作。
