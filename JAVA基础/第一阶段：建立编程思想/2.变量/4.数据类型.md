Java 的数据类型，它严格分为两大类：**基本数据类型（Primitive Types）** 和 **引用数据类型（Reference Types）**。

理解这两大类型，是理解 Java 内存管理（栈和堆）的关键。

---

## Ⅰ. 基本数据类型 (Primitive Data Types)

基本数据类型的特点是它们在内存中占用固定大小的空间，且变量直接存储着**实际的数值**，而不是地址。Java 一共提供了 **8 种**基本数据类型。

### A. 整数类型 (Integer Types)

用于存储整数值。它们都是有符号的（可以表示正数和负数）。

| 类型                  | 占用空间       | 取值范围              | 用途                                                  |
| :-------------------- | :------------- | :-------------------- | :---------------------------------------------------- |
| **`byte`**            | 8 位 (1 字节)  | **$-128$ 到 $127$**   | 节省空间，尤其在处理大型数组或文件流时。              |
| **`short`（短整型）** | 16 位 (2 字节) | $-32,768$ 到 $32,767$ | 较少使用，通常用于替代 `byte` 的更大范围。            |
| **`int`**             | 32 位 (4 字节) | **约 $\pm 20$ 亿**    | **最常用**的整数类型，默认的整数面值类型。            |
| **`long`（长整型）**  | 64 位 (8 字节) | 极大，约 $10^{18}$    | 需要存储比 `int` 更大的整数，常用于时间戳或文件大小。 |

> **注意：** 整数面值**默认为 `int` 类型**。如果需要表示 `long` 类型的字面量，必须在数字后面加上 `L` 或 `l`（推荐大写 `L`）。
> 示例: `long bigNum = 3000000000L;` > **提示：** 如果需要存放更大的数据，可以使用`BigInteger`类和`BigDecimal` 类

### B. 浮点类型 (Floating-point Types)

用于存储带有小数点的数值。

| 类型         | 占用空间       | 精度标准 | 用途                                   |
| :----------- | :------------- | :------- | :------------------------------------- |
| **`float`**  | 32 位 (4 字节) | 单精度   | 存储空间效率更高，但精度较低。         |
| **`double`** | 64 位 (8 字节) | 双精度   | **最常用**的浮点类型，提供更高的精度。 |

> **注意：** **浮点数面值默认为 `double` 类型**。如果需要表示 `float` 类型的字面量，必须在数字后面加上 `F` 或 `f`。如： `float pi = 3.14F;` 因为 java 比 C 更安全。
> **警告：浮点数存在精度缺失**，所以**通常情况下应该使用 `double` 类型**。如果需要精确地进行货币运算，应使用 `BigDecimal` 类。

> **浮点型常量有两种表示形式：**
>
> - 十进制：如 `5.12` `512.0f` `.512`（必须有小数点）
> - 科学计数法： 如 `5.12e2` `5.12E-2`

### C. 字符类型 (Character Type)

##### （1）基本用法

用于存储**单个字符**。

| 类型       | 占用空间       | 取值范围                                        | 特点                                                    | 用途                                                                                  |
| :--------- | :------------- | :---------------------------------------------- | :------------------------------------------------------ | :------------------------------------------------------------------------------------ |
| **`char`** | 16 位 (2 字节) | $0$ 到 $65535$（即 $2^{16} - 1$）之间的所有字符 | 采用 **Unicode** 编码标准，可以表示绝大多数语言的字符。 | 存储单个字符（可以放汉字），使用**单引号 `' '`** 包裹。双引号用于表示 `String` 字符串 |

> 示例: `char letter = 'A';` `char chinese = '中';`

##### （2）字符类型的数学特性

> 由于 char 在底层存储的是其字符对应的 Unicode 编码值（**一个 0-65535 之间的整数**），所以它**本质上就是一个整数类型**。因为数值特性，它可以：
>
> **A. 算术运算**：`char` 变量**可以直接参与算术运算**。在运算时，它会自动**提升为 `int` 类型**，参与运算的是其**码点值**。
> **B. 比较操作**：`char` 变量**可以直接使用比较运算符**`（>, <, ==）`进行比较，比较的是它们在 Unicode 表中的码点值。

> **拓展**：unicode 编码在转化成二进制过程中，有类似于数据类型的功能在告诉程序“如何处理这块数据”。

##### （3）特殊字符与表示法

另外，除了基本用法与赋值以外，char 类型允许通过**特殊的转义序列**和 **Unicode 编码**来表示难以直接输入的字符。

- **a. 转义序列**
  用于表示特殊的控制字符，用反斜杠 `\` 开头。

| 序列 | 含义            | 示例                     |
| :--- | :-------------- | :----------------------- |
| `\n` | 换行 (New Line) | `char newline = '\n';`   |
| `\t` | 制表符 (Tab)    | `char tab = '\t';`       |
| `\'` | 单引号字符      | `char quote = '\'';`     |
| `\\` | 反斜杠字符      | `char backslash = '\\';` |

- **b. Unicode 编码表示法**
  可以直接使用十六进制的 Unicode 编码值来赋值，因为编译器在编译的时候会先把内存中的程序解码，最终还原成 unicode 编码，然后基础上再转化成二进制的字节码文件。所以，不论是整数，字符还是 unicode 码来赋值，对于计算机来说都是一样的。因为：**“有关变量的问题其实就是内存的问题”，只要数据再内存中的存储形式和空间大小合适，就可以相互存储**

```java
char unicodeChar = '\u0041'; // '\u' 是前缀，0041 是 'A' 的十六进制码点，等价于 char unicodeChar = 65;
// unicodeChar 的值为 'A'
```

$$\text{'\u0041'} \xrightarrow{\text{编译器识别}} \text{十进制 65} \xrightarrow{\text{编译器转换}} \text{16 位二进制} \xrightarrow{\text{运行时分配}} \text{存储在变量 } unicodeChar \text{ 中}$$

### D. 布尔类型 (Boolean Type)

用于存储逻辑值。

| 类型          | 占用空间        | 取值                      | 用途                                   |
| :------------ | :-------------- | :------------------------ | :------------------------------------- |
| **`boolean`** | 独立于 JVM 实现 | **`true`** 或 **`false`** | 控制程序流程（例如 `if` 语句和循环）。 |

> **注意事项：不可以用 `0` 或者 `非0` 的整数代替 `true` 或者 `false`** ，这点和 C 语言不通

## Ⅱ. 引用数据类型 (Reference Data Types)

引用数据类型用于存储对象的地址，而不是实际数据。它们是 Java **面向对象**特性的基础。

### A. 核心特点

1.  **存储机制：** 引用变量在**栈**上存储着一个地址（**引用**或**指针**）。这个地址指向对象在**堆（Heap）**内存中的实际存储位置。
2.  **大小不定：** 它们的大小不是固定的，由它们引用的对象在运行时动态决定。
3.  **默认值：** 引用类型的默认值是 `null`（表示不指向任何对象）。

### B. 常见引用类型

1.  **类 (Classes)：** 除了 8 种基本类型之外的所有类，包括 Java 提供的内置类（如 `String`, `Scanner`, `Date`）和所有用户自定义的类。
2.  **接口 (Interfaces)：** 接口也是一种引用类型，变量可以引用实现了该接口的对象。
3.  **数组 (Arrays)：** 无论是基本类型的数组（如 `int[]`）还是引用类型的数组（如 `String[]`），数组本身都是对象，因此属于引用类型。

#### 示例：

```java
// 这是一个引用类型变量，它在栈上存储了一个地址
// 该地址指向堆上 String 对象的实际数据 "Hello"。
String greeting = "Hello";

// 这是一个引用类型变量，它指向一个自定义的 Person 对象。
Person student = new Person();
```

## Ⅲ. 基本类型 vs. 引用类型的关键区别

| 特性         | 基本数据类型 (Primitive)                    | 引用数据类型 (Reference)                        |
| :----------- | :------------------------------------------ | :---------------------------------------------- |
| **存储内容** | **实际值**                                  | **内存地址（引用）**                            |
| **内存位置** | 主要在 **栈 (Stack)** 内存中                | 引用本身在栈上，对象实体在 **堆 (Heap)** 内存中 |
| **默认值**   | 数字为 $0$，布尔为 `false`，字符为 `\u0000` | **`null`**                                      |
| **大小**     | 固定（如 `int` 始终是 4 字节）              | 不固定（取决于对象的大小）                      |
